Quick overview

These are the main areas of the system and what they are used for.

User Accounts

Church Admin creates accounts for Member, Treasurer, and Pastor.

Income Entry

Physical income is recorded through Unified Income Entry.

Assets & Banking

Configure bank settings, deposit cash, withdraw cash, and record direct bank transactions.

Budgeting & Requests

Create budgets, set allocations, request funds, release funds, approve requests, and liquidate expenses.

Role guide

Use this section to understand what each role normally does in the system.

Church Admin
Main controller of the system

Church Admin can access all major features. This role sets up the system, creates user accounts, configures categories, enables fraud detection, manages budgets, and oversees financial workflows.

Treasurer
Handles finance operations

Treasurer records income, manages bank-related finance entries, works on releases and liquidation, and helps keep accounting records complete and accurate.

Pastor
Approves requests

Pastor can review and approve requests, especially in fund request workflows under Execution and Payouts.

Member / Assigned Leader
Requests and role-based participation

Assigned leaders can request funds through Manage Request when they are authorized to do so. Members use the features available to their assigned role and access level.

Step-by-step workflow

This is the recommended order of work for setting up and using the system properly.

1

Create user accounts

Church Admin task
  • Church Admin creates user accounts for Member, Treasurer, and Pastor.
  • Each user should receive the correct role because system access depends on user type.
  • Make sure the users are linked to the correct church.
2

Configure categories and controls

Management & Financial Controls
  • Add Donation and Other Income categories here before recording those transactions.
  • Use this section to maintain the category structure of the system.
  • Enable fraud detection here if you want proof files to be required and validated during physical income entry.
3

Set up bank information

Assets & Banking → Bank Settings
  • Configure the church bank account first before using banking features.
  • Enter the correct bank name, account number, and account name.
  • Upload the required verification file if your setup requires account verification.
4

Add physical income

Unified Income Entry
  • Use Unified Income Entry to add physical income.
  • This includes entries such as tithes, offerings, donations, and other income depending on your setup.
  • If fraud detection is enabled, attach the correct proof file because the system may validate it before saving or review.
5

Use Assets & Banking properly

Assets & Banking
  • Use Deposit Cash to move physical cash into the bank.
  • Use Withdraw Cash to move money from bank to physical cash.
  • Use the bank movement feature for income or expenses that affect the bank directly.
  • For direct bank expenses, use the wording and option that matches your actual screen, usually Bank-paid Expense.
6

Create the budget structure

Optimization → Budget Management
  • Church Admin adds the budget name first under Budget Management.
  • This creates the budget record that will later receive allocations and requests.
  • Use clear budget names so leaders and treasurers can identify them easily.
7

Set budget allocation

Planning → Set Allocation
  • After creating the budget name, enter the budget amount in Set Allocation.
  • This is the planning step that determines how much is available for the budget.
  • Only proceed to requests and releases after allocation is properly set.
8

Request and approve funds

Execution & Payouts → Manage Request
  • Assigned leaders can request funds using Manage Request.
  • Pastor can review and approve each request.
  • This step helps control the release of budgeted funds before money is issued.
9

Release funds

Execution & Payouts → Release Funds
  • After approval, released funds are recorded in Release Funds.
  • This step documents the actual payout of requested money.
  • Treasurer should make sure the released amount matches the approved request.
10

Liquidate released expenses

Execution & Payouts → Budget Expense (Liquidate)
  • This feature helps the Treasurer liquidate expenses that were already released.
  • Use it to record how the released funds were spent.
  • Attach correct documents and expense details so reporting stays accurate.

Important reminders

These reminders help users avoid common mistakes during finance operations.

Fraud detection: If fraud detection is enabled in Management & Financial Controls, users must attach the correct proof file when adding physical income. Invalid or unclear files may stop the transaction from proceeding normally.
Categories first: Donation and Other Income categories should be created before users start encoding those transactions. If categories are missing, the user may not be able to classify the entry properly.
Budget order matters: The normal budgeting sequence is:

Budget Management → Set Allocation → Manage Request → Pastor Approval → Release Funds → Budget Expense (Liquidate)
